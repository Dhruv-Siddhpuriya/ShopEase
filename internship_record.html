ShopEase – Internship Daily Work Record

Project: ShopEase – Full Stack E-Commerce Web Application

Tech Stack: MongoDB · Express.js · React.js · Node.js (MERN Stack)

12 Weeks 60 Working Days MERN Stack Full Stack Development
Week 1 Project Planning & Environment Setup
Day 1Studied the project requirements and finalized the complete feature list for ShopEase including user authentication, product browsing, shopping cart, order management, and an admin panel for managing the platform.
Day 2Installed Node.js (v18), MongoDB Community Server, and VS Code on the development machine, then created the project folder structure separating frontend/ and backend/ directories with a root README.md.
Day 3Initialized the backend with npm init inside the backend/ folder and installed core packages: express, mongoose, dotenv, cors, bcryptjs, and jsonwebtoken.
Day 4Created server.js as the Express app entry point, configured middleware for JSON parsing and CORS, and connected to MongoDB using Mongoose with environment variables stored in a .env file.
Day 5Initialized the React frontend using npm create vite@latest, installed react-router-dom and axios, and verified both frontend and backend dev servers run simultaneously without conflict.
Week 2 User Authentication (Backend)
Day 1Designed and created the User Mongoose model in backend/models/User.js with fields: name, email, password (hashed), isAdmin (boolean, default false), and createdAt timestamp.
Day 2Built the registerUser controller that checks for duplicate emails, hashes the password using bcryptjs with a salt of 10 rounds, saves the new user to MongoDB, and returns user info with a JWT token.
Day 3Built the loginUser controller that finds a user by email, compares the entered password with the stored hash using bcrypt.compare(), and returns a signed JWT token valid for 30 days upon successful match.
Day 4Created authMiddleware.js with a protect middleware that reads the Bearer token from the Authorization header, verifies it using jwt.verify(), and attaches the decoded user to req.user.
Day 5Created an admin middleware that checks req.user.isAdmin and returns 403 Forbidden if not admin, then tested all auth endpoints using Postman with valid and invalid payloads.
Week 3 User Authentication (Frontend)
Day 1Created LoginPage.jsx and RegisterPage.jsx with controlled React form inputs for email, password, and name, including inline validation messages for empty fields and invalid email format.
Day 2Wired both forms to the backend using axios.post() — register hits /api/users/register, login hits /api/users/login — and handled API errors by displaying server error messages below the form.
Day 3Created AuthContext.jsx using React's createContext and useReducer to globally manage the userInfo state including token, name, email, and isAdmin flag across all components.
Day 4Built a PrivateRoute wrapper component in React Router that checks if the user is logged in via AuthContext and redirects unauthenticated users to /login with redirect state preserved.
Day 5Stored userInfo in localStorage so the session persists across page refreshes, implemented the logout function to clear context state and localStorage, and added a logout button in the Navbar.
Week 4 Product Management (Backend)
Day 1Designed the Product Mongoose model in backend/models/Product.js with fields: name, description, price, category, brand, countInStock, images (array), rating, and numReviews.
Day 2Created GET /api/products that fetches all products from MongoDB, supports an optional ?keyword= query for name-based search using a case-insensitive regex, and returns the full list as JSON.
Day 3Built POST /api/products (admin only) that accepts product data from the request body, creates and saves a new Product document to MongoDB, applying both protect and admin middlewares.
Day 4Implemented PUT /api/products/:id to find and update a product's fields from the request body, and DELETE /api/products/:id to remove it — both protected by protect and admin middlewares.
Day 5Created GET /api/products/:id to fetch a single product by ID for the details page, and added proper error handling for invalid or non-existent IDs using a custom async error handler middleware.
Week 5 Product Listing (Frontend)
Day 1Built HomePage.jsx with a full-width hero banner containing a tagline and CTA button, followed by a "Featured Products" section that fetches and renders the latest 8 products from /api/products using useEffect.
Day 2Created ProductsPage.jsx that fetches the full product list from the backend and displays them in a responsive CSS Grid layout, with a loading spinner during fetch and an error message if the API call fails.
Day 3Added a category filter sidebar on ProductsPage.jsx with buttons for each category that filter products client-side, and a search bar at the top that filters products by matching the keyword against product names.
Day 4Built ProductDetailsPage.jsx that fetches a single product using its ID from useParams(), and displays the product's image, name, price, description, rating, stock availability, and an "Add to Cart" button.
Day 5Added a sort dropdown with options Price Low to High, Price High to Low, and Newest First — implemented by sorting the fetched products array in state using JavaScript's .sort() method before rendering the grid.
Week 6 Shopping Cart & Order Flow (Backend)
Day 1Designed the Cart Mongoose model with a reference to the User and an array of cart items each containing productId, name, price, image, and qty, then built POST /api/cart to add or update items.
Day 2Implemented GET /api/cart to retrieve the logged-in user's cart by matching userId from req.user._id, and DELETE /api/cart/:itemId to remove a product using MongoDB's $pull operator.
Day 3Designed the Order Mongoose model with fields: user (ref to User), orderItems (array), shippingAddress (object with street, city, pincode), paymentStatus, and orderStatus.
Day 4Built POST /api/orders that creates a new order document, uses Product.findById() to deduct ordered quantities from countInStock using $inc, and then clears the user's cart from the database.
Day 5Implemented GET /api/orders/myorders to return all orders for the logged-in user sorted newest first, and GET /api/orders/:id to return a single order with full detail, both protected by protect middleware.
Week 7 Shopping Cart & Order Flow (Frontend)
Day 1Built CartPage.jsx that reads cart items from CartContext, displays each item with its image, name, unit price, and quantity selector (+ / - buttons), and calculates item subtotals and total cart price dynamically.
Day 2Connected Cart page buttons to backend API calls — PUT /api/cart/:itemId for quantity change and DELETE /api/cart/:itemId for removal — then refreshed the cart state in CartContext after each operation.
Day 3Created CheckoutPage.jsx with a shipping address form (street, city, state, pincode) and an order summary panel on the right showing all cart items, individual prices, and the final total before order confirmation.
Day 4Wired the "Place Order" button to call POST /api/orders with cart items and shipping address, then redirected to OrderSuccessPage.jsx showing the order ID and a success message on a 201 response.
Day 5Built OrderHistoryPage.jsx that fetches all orders from GET /api/orders/myorders and displays them with order ID, date, total, and a color-coded status badge (Processing / Shipped / Delivered) plus a "View Details" link.
Week 8 Admin Panel
Day 1Created AdminDashboard.jsx accessible only to admin users via an AdminRoute wrapper, fetched summary stats from the backend (total products, orders, users count) and displayed them in stat cards.
Day 2Built ManageProductsPage.jsx that fetches all products and renders them in a table with columns for image, name, price, category, stock, and action buttons (Edit / Delete), with Delete making a DELETE /api/products/:id call.
Day 3Implemented an Add/Edit Product form in the admin panel that pre-fills with existing data when editing and sends a PUT request on save, or a POST request when adding a new product.
Day 4Built ManageOrdersPage.jsx that fetches all orders from GET /api/orders (admin route) and displays them in a table, with a dropdown for each order to update its status via PUT /api/orders/:id/status.
Day 5Created ManageUsersPage.jsx that fetches all registered users from GET /api/users (admin route) and lists them with name, email, and admin status toggle calling PUT /api/users/:id to update isAdmin.
Week 9 Product Image Upload Feature
Day 1Installed multer in the backend using npm install multer, configured a storage engine in multerConfig.js to save files to the backend/uploads/ directory with unique filenames using Date.now() + extension.
Day 2Created uploadRoutes.js with a POST /api/upload endpoint using multer.array('images', 5) to accept up to 5 image files per request and returned saved file paths in the response.
Day 3Updated the Product Mongoose model to ensure the images field is an array of strings (file paths), and updated both POST and PUT product controllers to accept and store multiple image paths.
Day 4Updated ManageProductsPage.jsx to replace the image URL text input with a file picker (<input type="file" multiple />), and added thumbnail previews of selected images using URL.createObjectURL().
Day 5On form submission, first sent selected files to POST /api/upload using a FormData object with multipart/form-data, then used the returned image paths in the product save API call to store them with the product.
Week 10 Account Dashboard & User Profile
Day 1Designed AccountDashboard.jsx as a multi-section page with a sidebar navigation containing links to "My Profile", "My Addresses", and "My Orders", and a main content area rendering the selected section on click.
Day 2Built the "My Profile" section form with inputs for name, email, and new password (optional), wired to PUT /api/users/profile using the logged-in user's JWT token, and updated AuthContext with the new user info on success.
Day 3Added an addresses array field to the User Mongoose model (each with street, city, state, pincode), then created POST, PUT, and DELETE address APIs under /api/users/address.
Day 4Built the "My Addresses" section on the frontend with a list of saved addresses, an "Add New Address" form appearing on button click, and edit/delete options for each address — all wired to the backend APIs.
Day 5Added client-side form validation (empty field checks, password length), displayed success and error toast notifications using a custom Toast component after each API operation, and secured all account routes behind PrivateRoute.
Week 11 UI Polish, Navigation & Responsiveness
Day 1Redesigned Navbar.jsx with the ShopEase logo on the left, category navigation links in the center, and icons on the right for Cart (with item count badge from CartContext) and a User dropdown showing Profile, Orders, and Logout.
Day 2Chose a consistent design system — deep navy (#0f172a) as background, blue-purple gradient for accents, and "Inter" from Google Fonts as the global typeface — and applied these tokens across all pages via CSS variables in index.css.
Day 3Added CSS transition animations for all buttons (transform: scale(1.05) on hover), card hover box-shadow effects, and a smooth loading skeleton shimmer animation using @keyframes to replace plain loading spinners on product grids.
Day 4Wrapped all page layouts in responsive CSS Grid and Flexbox containers with media queries at 768px and 480px breakpoints, converted the product grid from 4-column to 2-column on tablet and 1-column on mobile, with a collapsible hamburger Navbar.
Day 5Added a BackButton component using useNavigate(-1) from React Router on all inner pages (Product Details, Order Details, Checkout), fixed broken internal links, and added a Footer.jsx with links, social icons, and copyright text.
Week 12 Testing, Bug Fixes & Final Submission
Day 1Performed complete end-to-end testing as a regular user — registered a new account, logged in, browsed products by category, searched for items, added to cart, proceeded to checkout, placed an order, and verified it appeared in Order History.
Day 2Logged in as an admin and tested all admin panel features — added a new product with multiple images, edited product details, deleted a product, updated an order's delivery status, and toggled a user's admin role — fixing all bugs found.
Day 3Fixed remaining issues: resolved cart count not updating after order placement by clearing CartContext, fixed Product Details page crash when images array was empty, and added a custom 404 page using React Router's * wildcard path.
Day 4Wrote the project README.md with sections covering project description, tech stack, full feature list, step-by-step local setup instructions, .env variable reference, and a complete list of all API endpoints with HTTP methods and access levels.
Day 5Ran a final full review of the entire application — verified all routes, confirmed admin and user access controls, ensured no console errors, checked mobile responsiveness, and prepared all project files and documentation for college submission.
